# うざい広告ジェネレーター

「うざいWeb広告」を安全に疑似体験できる、教育・ネタ目的の静的Webサイトです。
ポップアップ、全面広告、偽×ボタン、追従バナー、強制リダイレクトなど、
よくある「うざい広告」のパターンをパロディとして体験できます。

ビルド不要の純粋な HTML / CSS / バニラJS のみで構成されており、外部CDNや
外部ネットワークリクエストには一切依存していません。実際の広告配信・
トラッキング・外部サイトへの誘導・ファイルダウンロードは一切行いません。

## 特徴

- 11種類の「うざい広告」タイプを自由に組み合わせて体験できる
- 広告に使う画像・音声・キャッチコピーを自分でアップロード可能(任意)
- 記事は内蔵サンプル、または自分で貼り付けたテキストを使用可能
- すべての「危険な」遷移先はサイト内の `trap.html`(パロディページ)のみ
- レスポンシブ対応(スマホでの追従広告・全面広告なども体験可能)

## ファイル構成

```
uza-ad-generator/
├── index.html          … ジェネレーター画面(広告タイプ選択・素材アップロード)
├── experience.html      … 体験画面(記事+広告が発動する)
├── trap.html             … 偽×ボタン・強制リダイレクトの飛び先(パロディ警告ページ)
├── css/
│   └── style.css         … 全ページ共通スタイル
├── js/
│   ├── generator.js      … index.html用のロジック
│   ├── ads.js             … 11種類の広告タイプの実装(AdTypesレジストリ)
│   └── articles.js        … 内蔵サンプル記事データ
└── README.md
```

## 使い方(ローカル)

ビルドや依存関係のインストールは不要です。`index.html` をブラウザで開くだけで
動作します。

```
# 例: プロジェクトフォルダをローカルサーバーで配信する場合(任意)
cd uza-ad-generator
python -m http.server 8000
# ブラウザで http://localhost:8000 を開く
```

`file://` で直接 `index.html` を開いても動作しますが、ブラウザによっては
`sessionStorage` の扱いが制限される場合があるため、簡易ローカルサーバー
経由での閲覧を推奨します。

## GitHub Pagesへのデプロイ手順

1. このフォルダ(`uza-ad-generator`)の中身をGitHubリポジトリにpushする。
   - リポジトリのルートに置く場合はそのままpush
   - サブフォルダとして置く場合は、GitHub Pagesの公開対象フォルダ設定に合わせる
2. GitHubリポジトリの `Settings > Pages` を開く。
3. `Source` を `Deploy from a branch` に設定し、公開したいブランチ
   (通常は `main`)と、公開フォルダ(ルートに置いた場合は `/root`)を選択して保存する。
4. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` (サブフォルダに
   置いた場合は `https://<ユーザー名>.github.io/<リポジトリ名>/uza-ad-generator/`)
   でサイトが公開される。
5. `index.html` が入り口になっていることを確認する(ビルド不要・相対パスのみ
   なので追加設定は不要)。

## 広告タイプと実装の対応(概要)

各広告タイプは `js/ads.js` 内で「1タイプ=1関数」として実装され、
`AdTypes` レジストリ(`{ popup: initPopupAd, ... }`)経由で
`experience.html` のメインスクリプトから呼び出されます。
将来新しい広告タイプを追加する場合は、

1. `js/ads.js` に `init◯◯Ad(config, ctx)` 関数を追加
2. 同ファイル末尾の `AdTypes` に登録
3. `js/generator.js` の `AD_TYPE_DEFS` にチェックボックス定義を追加

するだけで拡張できます。

## 注意事項

- 本サイトはネタ・教育目的のパロディです。実在の企業名・商品名は使用していません。
- `trap.html` を含め、すべての遷移はサイト内のページにのみ限定されています。
- 実際のトラッキングコード・アナリティクス・外部リクエスト・ダウンロードを
  発生させる処理は一切含まれていません。
